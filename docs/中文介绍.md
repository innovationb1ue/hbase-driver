# HBase Python Driver - 纯 Python 原生 HBase 客户端库

## 简介

[hbase-driver](https://github.com/innovationb1ue/hbase-driver) 是一个纯 Python 实现的原生 HBase 客户端库，无需 Thrift 依赖。它实现了 HBase 的核心 RegionServer 和 Master RPC 协议，使 Python 程序能够直接与 HBase 集群交互，执行表操作和元数据操作。

- **GitHub**: https://github.com/innovationb1ue/hbase-driver
- **PyPI**: https://pypi.org/project/hbase-driver/

### 主要特点

- **纯 Python 实现**：无 Thrift 依赖，直接使用 HBase RPC 协议
- **性能优化**：内置连接池、区域缓存、批处理支持
- **完整 API**：支持所有核心 DML 操作（Put、Get、Delete、Scan）和 DDL 操作
- **过滤器支持**：完整的服务端过滤器实现，减少数据传输
- **上下文管理**：自动资源清理，类似 Java 的 try-with-resources
- **Java 兼容**：API 设计参考 HBase Java 驱动器，易于迁移

### 项目状态

- **集成测试**：77/77 测试通过（2026-02-21），使用自定义 3 节点 Docker 集群
- **Apache 许可**：Apache License 2.0

## 快速开始

### 基本连接和操作

```python
from hbasedriver.client.client import Client
from hbasedriver.hbase_constants import HConstants
from hbasedriver.operations.put import Put
from hbasedriver.operations.get import Get
from hbasedriver.operations.scan import Scan

# 初始化客户端
config = {
    HConstants.ZOOKEEPER_QUORUM: "localhost:2181"
}
client = Client(config)

# 获取表对象
table = client.get_table("default", "mytable")

# 写入数据
table.put(Put(b"row1").add_column(b"cf", b"col", b"value"))

# 读取数据
row = table.get(Get(b"row1"))
print(row.get(b"cf", b"col"))  # 输出: b"value"

# 扫描数据
with table.scan(Scan()) as scanner:
    for row in scanner:
        print(row.rowkey)

# 关闭客户端
client.close()
```

### 使用上下文管理器自动清理资源

```python
from hbasedriver.client.client import Client
from hbasedriver.hbase_constants import HConstants

# 使用上下文管理器 - 自动清理
with Client({HConstants.ZOOKEEPER_QUORUM: "localhost:2181"}) as client:
    with client.get_table("default", "mytable") as table:
        # 执行操作
        table.put(Put(b"row1").add_column(b"cf", b"col", b"value"))
    # 表和连接在这里自动关闭
```

## 核心 API

### Client - 客户端入口

`Client` 是 HBase 操作的主入口点，负责与集群的连接管理。

```python
from hbasedriver.client.client import Client
from hbasedriver.hbase_constants import HConstants

config = {
    HConstants.ZOOKEEPER_QUORUM: "localhost:2181"
}
client = Client(config)

# 获取 Admin 用于 DDL 操作
admin = client.get_admin()

# 获取 Table 用于数据操作
table = client.get_table("default", "mytable")

# 关闭客户端
client.close()
```

### Table - 表操作

`Table` 类提供表级别的数据操作。

```python
from hbasedriver.operations.put import Put
from hbasedriver.operations.get import Get
from hbasedriver.operations.delete import Delete
from hbasedriver.operations.scan import Scan

# Put - 插入或更新数据
table.put(Put(b"row1").add_column(b"cf", b"col", b"value"))

# Get - 获取单行数据
row = table.get(Get(b"row1"))

# Delete - 删除数据
table.delete(Delete(b"row1"))

# Scan - 扫描多行数据
with table.scan(Scan()) as scanner:
    for row in scanner:
        print(row.rowkey)
```

### Admin - DDL 操作

`Admin` 类提供数据定义语言（DDL）操作。

```python
from hbasedriver.model import ColumnFamilyDescriptor
from hbasedriver.table_name import TableName
from hbasedriver.client.admin import ColumnFamilyDescriptorBuilder

# 创建表
cf = ColumnFamilyDescriptorBuilder(b"cf").build()
admin.create_table(TableName.value_of(b"mytable"), [cf])

# 禁用表
admin.disable_table(TableName.value_of(b"mytable"))

# 启用表
admin.enable_table(TableName.value_of(b"mytable"))

# 删除表（必须先禁用）
admin.disable_table(TableName.value_of(b"mytable"))
admin.delete_table(TableName.value_of(b"mytable"))

# 截断表（删除所有数据但保留表结构）
admin.truncate_table(TableName.value_of(b"mytable"), preserve_splits=False)

# ⚠️ 重要：截断表后需要清除区域缓存
# table.invalidate_cache()
```

## 高级特性

### 过滤器（Filters）

驱动程序支持服务端过滤器，可在 RegionServer 上过滤数据，减少网络传输。

#### 基础过滤器

```python
from hbasedriver.filter import (
    PrefixFilter,          # 行键前缀过滤
    RowFilter,            # 行键比较过滤
    PageFilter,            # 限制返回行数
    KeyOnlyFilter,         # 仅返回行键
    FirstKeyOnlyFilter,    # 仅返回每行的第一列
)
from hbasedriver.filter.compare_operator import CompareOperator
from hbasedriver.filter.binary_comparator import BinaryComparator

# 前缀过滤
scan = Scan().set_filter(PrefixFilter(b"abc"))

# 行键过滤
scan = Scan().set_filter(
    RowFilter(CompareOperator.EQUAL, BinaryComparator(b"row1"))
)

# 分页过滤
scan = Scan().set_filter(PageFilter(100))

# 仅第一列（用于优化行计数查询）
scan = Scan().set_filter(FirstKeyOnlyFilter())
```

#### 组合过滤器

```python
from hbasedriver.filter import FilterList, FilterListOperator

# AND 逻辑 - 所有过滤器必须通过
scan = Scan().set_filter(
    FilterList(
        [PrefixFilter(b"row"), PageFilter(100)],
        FilterListOperator.MUST_PASS_ALL
    )
)

# OR 逻辑 - 任一过滤器通过即可
scan = Scan().set_filter(
    FilterList(
        [RowFilter(CompareOperator.EQUAL, BinaryComparator(b"row1")),
        [RowFilter(CompareOperator.EQUAL, BinaryComparator(b"row2"))],
        FilterListOperator.MUST_PASS_ONE
    )
)
```

#### 高级过滤器

```python
from hbasedriver.filter import (
    ColumnPrefixFilter,           # 列前缀过滤
    SingleColumnValueFilter,     # 单列值过滤
    MultipleColumnPrefixFilter,    # 多列前缀过滤
    ValueFilter,                 # 值过滤
    QualifierFilter,             # 限定符过滤
    FamilyFilter,                # 列族过滤
)

# 单列值过滤 - 类似 SQL WHERE 子句
scan = Scan().set_filter(
    SingleColumnValueFilter(
        column_family=b"cf",
        column_qualifier=b"status",
        compare_operator=CompareOperator.EQUAL,
        comparator=BinaryComparator(b"active"),
        filter_if_missing=False
    )
)

# 多列前缀过滤
scan = Scan().set_filter(
    MultipleColumnPrefixFilter([b"name_", b"addr_", b"email_"])
)
```

### 批处理（Batch Operations）

批处理可显著提高多行操作的效率。

```python
from hbasedriver.operations.batch import BatchGet, BatchPut

# 批量写入
bp = BatchPut()
bp.add_put(Put(b"row1").add_column(b"cf", b"col", b"value1"))
bp.add_put(Put(b"row2").add_column(b"cf", b"col", b"value2"))
results = table.batch_put(bp)

# 批量读取
bg = BatchGet([b"row1", b"row2", b"row3"]).add_column(b"cf", b"col")
results = table.batch_get(bg)

# 使用上下文管理器进行批处理
with table.batch(batch_size=100) as batch:
    batch.put(b"row1", {b"cf:col": b"value1"})
    batch.put(b"row2", {b"cf:col": b"value2"})
    # 退出上下文时自动执行所有操作
```

### CheckAndPut - 条件更新

原子性地检查并更新数据，类似数据库的 CAS（Compare-And-Swap）操作。

```python
from hbasedriver.operations.increment import CheckAndPut

# 检查并更新
cap = CheckAndPut(b"lock")
cap.set_check(b"cf", b"lock", b"")  # 检查锁是否为空
cap.set_put(Put(b"lock").add_column(b"cf", b"lock", b"held"))
success = table.check_and_put(cap)

if success:
    print("获取锁成功")
else:
    print("锁已被占用")
```

### Increment - 原子计数器

原子性地递增列值。

```python
from hbasedriver.operations.increment import Increment

inc = Increment(b"counter").add_column(b"cf", b"count", 1)
new_value = table.increment(inc)
print(f"计数器新值: {new_value}")
```

## 配置选项

| 配置键 | 默认值 | 说明 |
|---------|---------|------|
| `hbase.zookeeper.quorum` | 必需 | ZooKeeper 集群地址（逗号分隔） |
| `zookeeper.znode.parent` | `/hbase` | ZooKeeper 父节点路径 |
| `hbase.connection.pool.size` | 10 | 每个地址的最大连接数 |
| `hbase.connection.idle.timeout` | 300 | 空闲连接超时时间（秒） |

### 使用配置常量

```python
from hbasedriver.hbase_constants import HConstants

config = {
    HConstants.ZOOKEEPER_QUORUM: "localhost:2181",
    HConstants.CONNECTION_POOL_SIZE: 20,
    HConstants.CONNECTION_IDLE_TIMEOUT: 600,
}
client = Client(config)
```

### 使用字符串字面量（向后兼容）

```python
config = {
    "hbase.zookeeper.quorum": "localhost:2181",
    "hbase.connection.pool.size": 20
}
client = Client(config)
```

## 性能优化建议

### 1. 使用连接池

驱动程序内置连接池，自动复用连接到同一 RegionServer 的连接。

```python
config = {
    HConstants.CONNECTION_POOL_SIZE: 20,  # 增加连接池大小
    HConstants.CONNECTION_IDLE_TIMEOUT: 600,  # 延长空闲超时
}
client = Client(config)
```

### 2. 使用批处理

对于多行操作，使用批处理可显著提高性能。

```python
with table.batch(batch_size=1000) as batch:
    for i in range(1000):
        batch.put(f"row_{i}".encode(), {b"cf:col": b"value"})
```

### 3. 使用过滤器减少数据传输

在服务端过滤数据，减少网络传输量。

```python
# 仅返回需要的列
from hbasedriver.filter import ColumnPrefixFilter
scan = Scan().set_filter(ColumnPrefixFilter(b"required_prefix"))

# 限制返回行数
from hbasedriver.filter import PageFilter
scan = Scan().set_filter(PageFilter(100))
```

### 4. 指定列族和列

仅在需要时添加特定列，减少不必要的数据获取。

```python
# 仅获取需要的列
get = Get(b"row1").add_column(b"cf", b"col1")

# 仅扫描特定列族
scan = Scan().add_column(b"cf", b"col1")
```

### 5. 清理区域缓存

在表结构变更（如截断）后，清除区域缓存以使用新的区域信息。

```python
# 截断表后清除缓存
admin.truncate_table(table_name, preserve_splits=False)
table.invalidate_cache()  # 重要：清除过时的区域缓存
```

## Java 兼容性

本驱动程序设计对 HBase Java 开发者友好。API 设计参考了 HBase Java 驱动器。

| Java 驱动器 | Python 驱动器 |
|-------------|---------------|
| `Connection connection = ConnectionFactory.createConnection(config)` | `client = Client(config)` |
| `Table table = connection.getTable(TableName.valueOf("mytable"))` | `table = client.get_table("default", "mytable")` |
| `table.put(new Put(Bytes.toBytes("row1"))...` | `table.put(Put(b"row1")...` |
| `Result result = table.get(new Get(Bytes.toBytes("row1"))...` | `row = table.get(Get(b"row1")...` |
| `try (ResultScanner scanner = table.scan(...))` | `with table.scan(...) as scanner:` |
| `try (Connection conn = ...)` | `with Client(config) as client:` |

### 配置常量对应

Java 的 `HConstants` 可用作 `hbase_constants.HConstants`：

| Java | Python |
|------|--------|
| `HConstants.ZOOKEEPER_QUORUM` | `HConstants.ZOOKEEPER_QUORUM` |
| `HConstants.ZOOKEEPER_ZNODE_PARENT` | `HConstants.ZOOKEEPER_ZNODE_PARENT` |
| 自定义连接池大小 | `HConstants.CONNECTION_POOL_SIZE` |
| 自定义空闲超时 | `HConstants.CONNECTION_IDLE_TIMEOUT` |

## 常见问题

### 问题：截断表后数据仍然存在

**原因**：截断表后，表的区域缓存仍然包含旧的区域信息。

**解决方案**：调用 `table.invalidate_cache()` 清除过时的缓存。

```python
admin.truncate_table(table_name, preserve_splits=False)
table.invalidate_cache()  # 清除过时的区域缓存
```

### 问题：连接超时

**原因**：ZooKeeper 或 HBase 集群未启动，或网络连接问题。

**解决方案**：
1. 检查 ZooKeeper 是否运行
2. 检查 HBase Master 是否运行
3. 验证配置中的 ZooKeeper 地址正确
4. 增加超时时间

```python
config = {
    HConstants.ZOOKEEPER_QUORUM: "localhost:2181",
    HConstants.ZOOKEEPER_SESSION_TIMEOUT: 30000,  # 增加会话超时
}
```
